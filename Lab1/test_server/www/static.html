📄 静态HTML页面缓存测试

✅ 这是一个静态HTML文件

Nginx会自动为静态文件添加 Last-Modified 头部（基于文件修改时间）。

测试步骤：

  1. 首次访问：查看代理服务器日志
    • 应显示：[缓存] 未找到缓存
    • 应显示：[响应] Last-Modified: ...
    • 应显示：[缓存] 已保存缓存
  2. 刷新页面（Ctrl+R 或 F5）：
    • 应显示：[缓存] 找到缓存
    • 应显示：[缓存] 添加 If-Modified-Since
    • 应显示：[响应] 状态码: 304
    • 应显示：[缓存] 服务器返回304 Not Modified，缓存仍然有效
  3. 检查缓存目录：
    • 在 ./cache 目录下应该有一个 .cache 文件

⚠️ 注意事项

  • 确保浏览器使用普通刷新（F5），不要使用强制刷新（Ctrl+F5）
  • 强制刷新会添加 Cache-Control: no-cache 头，绕过缓存
  • 可以打开浏览器开发者工具（F12）查看网络请求

页面信息：

  • 文件类型：静态HTML
  • 缓存支持：✅ 是
  • Last-Modified：由Nginx根据文件修改时间自动添加
